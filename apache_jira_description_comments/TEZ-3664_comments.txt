I think the key take-away is that tests are writing outside of the test directory and in this case the '/tmp' directory. Many machines may have a temp cleaner that deletes temporary files written to this directory. A fix is to write test data within the repository as does Apache Hadoop to keep better control of the files under test.

001 patch address the lack of the tests relying on test.build.data and falling back to /tmp/ if not specified. This will handle a majority of test cases explicitly writing to tmp directory that could be subject to tmp dir cleaner.
