An application should never be accepted and rejected at the same time. The reject is caused by an auto generated event as a result of an other event coming in.
The base of the auto generated event is the APP_NEW_SAVED event which had a null placement context which would reject the application. If that reject is processed before the correct added app event is processed it could cause all kinds of issues.

Adding a placement context when we create the app to prevent a reject message from being created and cleanup the test.
