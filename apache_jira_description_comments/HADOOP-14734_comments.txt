A couple things to note about the patch:
 * The document based Dynamo API does not appear to expose access to the tagging API so we need to keep track of the {{AmazonDynamoDB}} object now. 
 * Each tag needs to be its own property in the config xml since commas are allowed in the keys and values of a tag
 * Tags are only added to a table when it is created and there is no retry logic
 * The test must connect to "real DynamoDB" instead of the in-memory version we use for some other tests, so it has its own class (since the in-memory version does not appear to support tagging)
 * The test assumes table creation privileges and that {{S3GUARD_DDB_REGION_KEY}} is being provided
