That shouldn't be an issue. Are you using a specific version of Hive, or just the master branch? IIRC this is actually a regression, it was working a few months ago. If you are using Hive 2.1.0, there shouldn't be any compatibility issues if this patch goes into 2.1.1 or 2.2.0.

Unless I'm mistaken, the S3 SDKs don't really have a concept of drivers. Connections to S3 from Hive are done via the {{S3AFileSystem}} class (which extends {{FileSystem}}, the HDFS equivalent is {{DistributedFileSystem}}). The {{S3AFileSystem}} connects to S3 using the S3 SDKs. These SDKs connect to S3 via HTTP. So the {{S3AFileSystem}} is going to use whichever AWS SDK it sees on the classpath. After this patch, it will use the SDK that is ships with (version 1.7.6).

The way shading works, there will be two AWS SDKs on the classpath, but one of them will be moved to the package {{org.apache.hive.druid.com.amazonaws}} so the {{S3AFileSystem}} will never see it. The {{S3AFileSystem}} will only look for an SDK under the package {{com.amazonaws}}.
